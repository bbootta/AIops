# limitation_and_risk.md

본 도구의 한계와 잘못 사용했을 때의 위험을 명시한다.

---

## 1. 도구의 한계

- 정량 지표는 모형 적합성에 대한 **참고 정보**다. 단독으로 적합/부적합을 결정하지 않는다.
- 데이터 정의(부도, 회수, 한도, 관측창)가 일관되지 않으면 모든 지표가 오해를 일으킬 수 있다.
- LDP, 신규 모형, 짧은 관측 기간은 정상적인 결과 해석이 어렵다.
- VIF / condition index / 잔차 진단은 **기본 진단**이며, 구조적 모형 적합성은 별도 검토가 필요하다.
- PSI bin 정의, calibration bucket 정의는 결과를 크게 좌우한다.

## 2. 사용상 위험

- 임계값을 임의 완화/강화하면 결과 해석이 무의미해진다.
- 운영 데이터를 본 저장소에 저장하면 즉각 정보보안 사고가 된다.
- 결과를 단정형으로 해석하여 외부에 인용하는 행위는 검증팀 의견의 신뢰도를 훼손한다.
- 시나리오 적용 결과만 보고 stress test 결론을 내려서는 안 된다.

## 3. 운영상 위험

- 본 에이전트는 운영계와 어떠한 형태로도 통신하지 않는다.
- `git push`, 배포, 외부 메일/메신저 전송은 자동 수행되지 않는다.
- 인간 검증자의 명시적 승인 없이는 의견 확정·정책 변경이 불가하다.
