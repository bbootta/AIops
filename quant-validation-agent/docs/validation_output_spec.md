# validation_output_spec.md

표준 검증 결과 출력 형식.
모든 검증 산출물은 본 형식을 따른다.

---

## 1. 검증 요약
- 모형명
- 모형 유형
- 검증 구분 (개발/운영/모니터링/변경/챌린저/시나리오)
- 데이터 기준일
- 표본 수
- 이벤트 수 (부도건수 등)
- 주요 결론 (단정 금지, 초안)
- RAG 상태 (Green/Yellow/Red/Gray)

## 2. 입력 데이터 점검

| 항목 | 결과 | 비고 |
|---|---|---|
| 필수 컬럼 | Pass/Fail | |
| 결측 | Pass/Warning/Fail | |
| 중복 | Pass/Warning/Fail | |
| 표본 수 | Pass/Warning/Fail | |
| 이벤트 수 | Pass/Warning/Fail | |
| 개발/운영 구분 | Pass/Fail | |
| 누수 가능성 | Pass/Warning/Fail | |

## 3. 주요 지표

| 지표 | 값 | 기준 | 상태 | 해석 |
|---|---:|---:|---|---|

## 4. 세부 분석
- 변별력
- 안정성
- 보정력
- 표본 적정성
- 시나리오 정합성
- 회귀진단
- 등급별 특이사항

## 5. 이상 징후

| 이슈 | 심각도 | 근거 | 원인 후보 | 추가 확인 |
|---|---|---|---|---|

## 6. 한계
- 데이터 한계
- 방법론 한계
- 표본 한계
- 해석상 주의사항

## 7. 검증 의견 초안
정량 결과에 근거한 **초안**. "적합/부적합 확정" 금지.

## 8. 추가 확인사항
- 데이터 담당 확인 필요
- 모형 개발부서 확인 필요
- 기준값/정책 확인 필요
- 재분석 필요사항

## 9. 감사추적
- 실행 파일
- 생성 파일
- 테스트 결과
- 로그 위치
- change_manifest 기록 여부
