# 검증 결과 예시

본 양식은 표준 검증 리포트 형식이다.
모든 결과는 정량 결과에 근거한 **초안**이며, 적합/부적합을 단정하지 않는다.

---

## 1. 검증 요약
- 모형명: 개인고객 신용평가모형 v3.2
- 모형 유형: 스코어링
- 검증 구분: 운영 검증
- 데이터 기준일: 2025-04-30 (운영) / 2025-01-31 (개발 비교)
- 표본 수: 20 (예시)
- 이벤트 수: 8 (부도)
- 주요 결론: 개발 대비 운영 표본의 등급 분포 차이는 제한적이며 변별력은 유지되는 것으로 관측됨. 운영 정책 임계값 확인 필요.
- RAG 상태: Yellow (표본 부족 사유)

## 2. 입력 데이터 점검

| 항목 | 결과 | 비고 |
|---|---|---|
| 필수 컬럼 | Pass | customer_id/score/target/obs_date/dataset 존재 |
| 결측 | Pass | 결측 없음 |
| 중복 | Pass | snapshot 단위 중복 없음 |
| 표본 수 | Warning | 예시 데이터 (20건) |
| 이벤트 수 | Warning | 부도 8건 |
| 개발/운영 구분 | Pass | dataset 컬럼 정상 |
| 누수 가능성 | Pass | 사후정보 의심 컬럼 없음 |

## 3. 주요 지표

| 지표 | 값 | 기준 | 상태 | 해석 |
|---|---:|---:|---|---|
| KS | 0.80 | ≥0.40 (예시) | Green | 변별력 강함 |
| AUROC | 0.92 | ≥0.75 (예시) | Green | 변별력 강함 |
| AR | 0.84 | ≥0.50 (예시) | Green | 변별력 강함 |
| PSI(grade) | 0.04 | <0.10 (예시) | Green | 분포 안정 |

## 4. 세부 분석
- 변별력: KS/AUROC/AR 모두 임계 충족 (예시 임계 기준)
- 안정성: 등급 분포 PSI 작음
- 보정력: 본 예시는 측정 대상 외 (PD가 아닌 score)
- 표본 적정성: 예시 표본이라 Yellow
- 시나리오 정합성: N/A (스코어링)
- 회귀진단: N/A
- 등급별 특이사항: 등급 E의 default rate 100%는 표본 수 부족 가능성

## 5. 이상 징후

| 이슈 | 심각도 | 근거 | 원인 후보 | 추가 확인 |
|---|---|---|---|---|
| 표본 수 부족 | Yellow | 20건 | 예시 데이터 | 실제 운영 표본으로 재실행 |

## 6. 한계
- 데이터 한계: 예시 표본 20건
- 방법론 한계: 정량 지표만으로 적합성 단정 불가
- 표본 한계: 등급별 표본이 매우 적음
- 해석상 주의사항: 임계값은 정책 미확보 시 Gray 처리 권장

## 7. 검증 의견 초안 (단정 금지)
운영 표본의 변별력은 유지되는 것으로 관측되나, 표본 수가 매우 적어 결과 일반화에 한계가 있다.
정책 임계값과 실제 운영 표본 확보 후 재검증이 필요하다.

## 8. 추가 확인사항
- 모형 개발부서: 등급 정의 변경 여부 확인
- 데이터 담당: 운영 표본 누적 기간 및 segment 정의 확인
- 정책 담당: KS/AUROC/AR/PSI 적용 임계 정책 확인

## 9. 감사추적
- 실행 파일: `tools/metric_ks_auc_ar.py`, `tools/metric_psi.py`, `tools/binning_stability.py`
- 생성 파일: `reports/credit_score_v3_2_2025Q2_summary.md` (예)
- 테스트 결과: `pytest -q` PASS
- 로그 위치: `logs/run_*.json`
- change_manifest 기록: `harness/change_manifest.json`에 1건 (예시)
