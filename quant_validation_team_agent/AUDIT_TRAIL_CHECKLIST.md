# AUDIT_TRAIL_CHECKLIST

## 1. 목적

양적검증 팀 에이전트 운영 과정의 입력, 판단 근거, 계산엔진 결과, 정책 버전, 인간 검토 결과를 재현 가능하게 남기기 위한 감사추적 체크리스트다.

## 2. 감사추적 원칙

- 모든 case는 case_id와 request_id를 가진다.
- 입력자료, 계산엔진 결과, 정책문서에는 버전 또는 해시를 기록한다.
- LLM이 직접 수치 계산을 하지 않았음을 명시한다.
- 판정 후보와 최종 조직 결정은 분리 기록한다.
- Yellow / Red / Gray에는 Action Notice ID를 연결한다.

## 4. Agent별 중간 산출물 목록

- Intake Summary
- Classification Memo
- Data Readiness Result
- Method Plan
- Calculation Evidence Log
- Result Review Memo
- Policy Mapping
- Judgement Memo
- Draft Report
- Action Notice
- Audit Pack
- Final Decision Record

## 5. 재현성 확보 기준

- 입력 데이터 파일명, 저장 위치, 기준일, 버전 또는 해시를 기록한다.
- 계산엔진 실행 ID, 실행일시, 파라미터 파일, 코드/배치 버전을 기록한다.
- 정책문서의 문서번호, 버전, 승인일, owner를 기록한다.
- 에이전트 응답의 생성일시와 사용한 입력자료 목록을 기록한다.
- 인간 검증자 검토 의견과 공식 조직 결정은 에이전트 판정 후보와 별도 항목으로 남긴다.

## 6. 운영상 가정

- 감사추적 저장소는 GRC, 문서관리시스템, 모델관리시스템 또는 내부 승인 시스템일 수 있다.
- 보존기간과 접근권한은 은행 내부 규정에 따른다.
