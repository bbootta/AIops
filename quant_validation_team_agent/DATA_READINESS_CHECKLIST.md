# DATA_READINESS_CHECKLIST

## 1. 목적

검증 전 데이터, 권한, lineage, 증적 준비 상태를 점검하여 정량결과 해석의 전제조건을 확인한다. 데이터 준비성이 부족하면 정량 결론을 내리지 않고 Gray 후보로 전환한다.

## 2. Gray 강제 조건

다음 중 하나라도 충족하면 기본 판정 후보는 Gray다.

- 데이터 부족
- 표본 부족
- 권한 부족
- lineage 불명확
- 증적 부족
- 정책 미정의

|---|---|---|---|---|---|
| 데이터 소유부서 | 데이터 owner 명세, 승인 메일 또는 시스템 권한 | 예/아니오 |  | 불명확 시 Gray | owner 확인 요청 |

## 4. 판정 예시와 Gray 사례 라이브러리

| 사례명 | 입력상황 | 사용 가능한 증적 | 부족한 증적 | 판정 후보 | Action Notice | 인간 검증자 확인사항 |
|---|---|---|---|---|---|---|
| 정책 미정의 | 신규 거시변수 검증 기준이 공식 정책에 없음 | 변수 후보 설명 | 공식 검증 기준 | Gray | 필요 | 정책 담당부서 기준 수립 여부 |
| 데이터 부족 | PD 검증 대상 기간 중 일부 월 데이터 누락 | 일부 월 데이터 | 누락 월 원천자료 | Gray | 필요 | 재추출 또는 제외 승인 가능 여부 |
| 표본 부족 | 특정 등급 default 표본 부족 | segment별 건수 | 확장 표본 또는 pooling 기준 | Gray | 필요 | 기간 확장, pooling 정책 적정성 |
| 권한 부족 | 담보 회수자료 접근권한 없음 | 요청서 | 회수자료 또는 대체 증적 | Gray | 필요 | 권한 승인 또는 masking 대안 |
| lineage 불명확 | 계산엔진 입력 데이터와 원천계 매핑 불명확 | 최종 입력 파일 | ETL 경로, 추출쿼리 | Gray | 필요 | 원천-중간-최종 흐름 재현성 |
| 증적 부족 | 결과 숫자는 있으나 실행 로그와 파라미터 없음 | 결과 파일 | 로그, 버전, 파라미터 | Gray | 필요 | 재실행 가능성 및 공식 결과 여부 |

## 5. 운영상 가정

- 데이터 품질 자체를 수치로 재계산하지 않고, 공식 품질 리포트와 계산엔진 결과를 검토한다.
- masking 또는 익명화 데이터 사용 시 원천과의 일관성 증적이 별도로 필요하다.

## 6. 공통 하네스 확인

- LLM은 결측률, 표본 수, 이상치 영향을 직접 재계산하지 않고 공식 데이터 품질 리포트와 계산엔진 결과를 참조한다.
- 데이터 준비성이 충분하고 중대 이슈 후보가 없더라도 Green은 최종 승인으로 해석하지 않는다.
- Yellow / Red / Gray가 필요한 데이터 이슈는 반드시 Action Notice로 연결한다.
