# JUDGEMENT_POLICY_TEMPLATE

## 1. 목적

검증 결과를 Green / Yellow / Red / Gray 중 하나의 판정 후보로 문서화하기 위한 정책 템플릿이다. 본 템플릿은 최종 승인 기준이 아니라 인간 검증자와 공식 조직의 판단을 지원하는 실무 기준이다.

## 2. 판정 라벨 정의

| 라벨 | 정의 | 후속조치 |
|---|---|---|
| Green | 현재 제공된 공식 증적과 계산엔진 결과 기준 중대한 이슈 후보가 발견되지 않은 상태. 최종 승인 아님. | 제한사항과 근거 기록, 인간 검토 |
| Yellow | 보완 필요, 제한적 이슈, 일부 segment 또는 문서 공백이 있으나 즉시 중대 결함으로 단정하기 어려운 상태. | Action Notice 생성 |
| Red | 중대한 결함, 정책 위반, 결과 신뢰성 훼손 가능성, 공식 보고 영향 가능성이 큰 상태. | Action Notice 생성 및 에스컬레이션 후보 표시 |
| Gray | 정책 미정의, 데이터 부족, 표본 부족, 권한 부족, lineage 불명확, 증적 부족 등 판단 불가 상태. | Action Notice 생성 및 판단 보류 |

Amber라는 용어는 판정 라벨로 사용하지 않는다. 입력 요청에 해당 용어가 포함되어도 Green / Yellow / Red / Gray 중 하나로 재분류하거나 Gray로 판단 보류한다.

## 3. 판정 후보 작성 양식

```markdown
## 판정 후보
- case_id:
- validation_object_type:
- provisional_judgement: Green | Yellow | Red | Gray
- 판정 근거:
- 사용한 정책문서 및 버전:
- 계산엔진 결과 참조:
- 데이터 기준일:
- 증적 공백:
- 제한사항:
- 인간 검증자 확인 필요사항:
- 공식 조직 최종 결정란:
```

## 4. 판정별 필수 기준

### Green

- 계산엔진 결과가 존재하고, 공식 증적 및 정책 기준과 중대한 충돌이 없어야 한다.
- 데이터, lineage, 권한, 재현성 증적이 충분해야 한다.
- Green은 최종 승인 또는 무결성 보증이 아니며 인간 검증자 검토가 필요하다.

### Yellow

- 보완 필요 사항이 있으나 사용 가능한 증적이 일부 존재하고 영향 범위가 제한적일 수 있다.
- 조치계획, 보완자료, 재검증 조건을 Action Notice에 기록한다.

### Red

- 정책 위반, 중대한 데이터 오류, 잘못된 모형/파라미터 버전 사용, 공식 보고 영향 가능성이 있는 경우다.
- Action Notice에 에스컬레이션 후보, 임시 사용 제한 필요 여부, 재산출 조건을 기록한다.

### Gray

- 정책 미정의, 데이터 부족, 표본 부족, 권한 부족, lineage 불명확, 증적 부족으로 판단이 불가능한 경우다.
- 결론을 내리지 않고 필요한 증적과 담당 부서를 Action Notice로 요청한다.

## 5. 판정 예시와 Gray 사례 라이브러리

| 사례명 | 입력상황 | 사용 가능한 증적 | 부족한 증적 | 판정 후보 | Action Notice | 인간 검증자 확인사항 |
|---|---|---|---|---|---|---|
| 정책 미정의 | 신규 거시변수 검증 기준이 공식 정책에 없음 | 변수 설명서 | 공식 검증 기준 | Gray | 필요 | 정책 기준 수립 여부 |
| 데이터 부족 | PD 검증 대상 기간 일부 월 누락 | 일부 월 산출물 | 누락 월 데이터 | Gray | 필요 | 제외 가능 여부와 재추출 필요성 |
| 표본 부족 | 특정 등급 default 표본 부족 | 등급별 건수 | 확장 표본 또는 pooling 기준 | Gray | 필요 | pooling 정책 사용 가능 여부 |
| 권한 부족 | 담보 회수자료 접근권한 없음 | 접근 요청 기록 | 회수자료 | Gray | 필요 | 권한 승인 또는 대체 증적 |
| lineage 불명확 | 원천계와 계산엔진 입력 매핑 불명확 | 최종 결과 | ETL 경로, 추출쿼리 | Gray | 필요 | lineage 재구성 가능 여부 |
| 증적 부족 | 결과 숫자는 있으나 실행 로그 없음 | 결과 파일 | 로그, 파라미터, 버전 | Gray | 필요 | 공식 산출물 인정 가능 여부 |
| 제한적 결측 | 일부 비핵심 필드 결측이나 승인된 처리정책 존재 | 결측 리포트, 정책 | 영향분석 보완자료 | Yellow | 필요 | 보완 후 Green 가능 여부 |
| 중대 버전 오류 | 미승인 모형 버전으로 결과 산출 | 실행 로그 | 승인 버전 결과 | Red | 필요 | 재산출 및 보고 영향 여부 |

## 6. 운영상 가정

- 판정은 후보이며, 공식 결정은 인간 검증자와 승인 조직이 기록한다.
- 정책 임계값이 문서에 없으면 에이전트가 임계값을 만들지 않는다.

## 7. LLM 하네스 확인

- LLM은 판정 후보를 문서화할 뿐 직접 수치 계산이나 최종 승인 판단을 수행하지 않는다.
- 계산엔진 결과와 공식 증적이 없으면 정량 결론을 내리지 않고 Gray로 판단 보류한다.
- 인간 검증자와 공식 조직은 에이전트 판정 후보를 채택, 수정, 반려할 수 있다.
