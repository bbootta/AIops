# QUANT_VALIDATION_METHOD_GUIDE

## 1. 목적

검증대상별 정량검증 방법 후보, 필요한 계산엔진 산출물, 해석 포인트, 흔한 오류, Gray 조건을 정의한다. 이 문서는 계산방법을 설명하지만 LLM이 직접 계산하지 않는다는 통제를 전제로 한다.

## 2. 공통 통제

- LLM은 직접 수치 계산, 재계산, 추정 계산, 임계값 산출을 수행하지 않는다.
- 계산엔진 결과, 승인된 리포트, 공식 데이터 증적 없이 정량 결론을 내리지 않는다.
- 수치 임계값은 정책문서 또는 승인된 계산엔진 설정 없이는 확정하지 않는다.
- 판정 라벨은 Green, Yellow, Red, Gray만 허용한다.
- 정책 미정의, 데이터 부족, 표본 부족, 권한 부족, lineage 불명확, 증적 부족은 Gray로 분류한다.

## 3. 신용평가모형 검증

| 방법 후보 | 목적 | 필요한 계산엔진 산출물 | 해석 포인트 | 흔한 오류 | Gray 조건 |
|---|---|---|---|---|---|
| 변별력 검토 | 우량/불량 차별 능력 확인 | AUROC, Gini, KS 등 승인 지표 리포트 | 정책 임계값 대비 결과, segment별 차이 | 표본기간 불일치, default 정의 혼용 | 계산결과 또는 임계값 부재 |
| 안정성 검토 | 모집단 변화와 모형 적용 안정성 확인 | PSI/CSI 등 안정성 리포트 | 급격한 분포 변화 여부 | 개발표본과 운영표본 비교 기준 불명확 | 모집단 lineage 불명확 |
| 보정 검토 | 예측위험과 관측위험의 정합성 확인 | calibration table, backtesting 결과 | 등급별 과소/과대 추정 여부 | 낮은 표본 segment 과해석 | 표본 부족 |
| 등급전이 검토 | 등급 이동의 합리성 확인 | transition matrix | 비정상 전이, 특정 등급 쏠림 | 정책상 관찰기간 불일치 | 관찰기간 정책 미정의 |
| override 검토 | 수동 조정의 정책 준수 확인 | override 건수·사유 리포트 | 사유코드, 승인권자, 성과 차이 | 증적 없는 override | override 증적 부족 |
| cut-off 검토 | 승인 기준과 리스크 기준 연결성 확인 | cut-off별 성과 리포트 | 승인 정책과 실제 적용 일치성 | 최신 정책 버전 미반영 | cut-off 승인문서 부재 |

## 8. 운영상 가정

- 계산엔진은 은행이 승인한 통계 패키지, 배치 시스템, 검증 플랫폼 또는 공식 BI 산출물을 포함한다.
- 에이전트는 계산엔진 결과의 출처·버전·제한사항을 검토하지만 결과값을 수정하지 않는다.

## 9. Action Notice 및 인간 검토 연결

- 검증방법 후보 선정 중 정책 미정의, 데이터 부족, 표본 부족, 권한 부족, lineage 불명확, 증적 부족이 확인되면 Gray 후보와 Action Notice를 생성한다.
- 계산엔진 결과가 정책 기준과 충돌하거나 중대한 결함 후보가 있으면 Yellow 또는 Red 후보와 Action Notice를 생성한다.
- 모든 방법 선택과 결과 해석은 인간 검증자 검토를 거쳐야 하며, 에이전트는 최종 승인 판단을 대체하지 않는다.
