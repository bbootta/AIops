# REG_CHANGE_CANDIDATE_CONTROL

## 1. 목적

규제 변화, 감독 지침, 내부 정책 변경 가능성이 양적검증 업무에 미치는 영향을 후보 수준에서 분석하고 후보 검증통제를 제안하기 위한 프로세스다.

## 2. 핵심 원칙

- 규제 변화는 자동 반영하지 않는다.
- LLM은 후보 영향분석과 후보 검증통제 제안까지만 수행한다.
- 공식 정책 반영, 내부 기준 변경, 모델 변경, 보고 변경은 인간 담당자 및 승인 조직이 수행한다.
- 규제 원문, 감독기관 공문, 내부 법무/컴플라이언스 검토가 공식 근거다.
- 정책 미정의 또는 해석 불명확 상태는 Gray로 기록한다.

## 3. 프로세스

| 단계 | 활동 | 산출물 | 금지사항 |
|---|---|---|---|
| 1. 규제 원문 수집 | 원문 출처, 발행일, 시행일, 적용 범위 기록 | Regulation Source Log | 비공식 요약만으로 판단 금지 |
| 2. 변경 요약 | 변경 조항, 삭제/신설 기준, 전환기간 요약 | Change Summary | 법적 해석 확정 금지 |
| 3. 영향 후보 식별 | 관련 모델, 파라미터, 보고서, 데이터, 프로세스 매핑 | Impact Candidate Map | 자동 업무 변경 금지 |
| 4. 후보 통제 제안 | 추가 체크리스트, 증적 요구, 계산엔진 산출물 요구 제안 | Candidate Control Proposal | 즉시 적용 지시 금지 |
| 5. 담당부서 검토 요청 | 정책, 법무, 리스크관리, 데이터 owner 검토 요청 | Review Request | 승인 생략 금지 |
| 6. 승인 전 보류 | 공식 승인 전 운영 반영 금지 문구 기록 | Pending Decision Log | 자동 반영 금지 |

## 5. 운영상 가정

- 감독규정과 내부 정책 간 충돌 또는 해석 공백은 Gray로 관리한다.
- 외부 규제 모니터링 도구가 제공하는 요약은 참고자료이며 원문 확인이 필요하다.

## 6. 판정 라벨 연결

- 규제 변화 후보 영향분석의 산출물에도 Green / Yellow / Red / Gray 외 판정 라벨을 사용하지 않는다.
- 공식 정책 반영 근거가 충분하지 않거나 해석이 불명확하면 Gray로 기록하고 Action Notice 또는 검토 요청을 발행한다.
- 규제 변화로 중대한 정책 위반 가능성이 확인되면 Yellow 또는 Red 후보로 에스컬레이션하되, 자동 반영은 금지한다.
