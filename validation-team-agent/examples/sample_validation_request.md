# Sample Validation Request

다음 양식을 채워서 오케스트레이터에 검증을 요청한다.

```
[검증 대상]
모형명: 개인신용평가모형 v3.2
모형군: 신용평가모형
사용 목적: 가계대출 신청 평가

[검증 목적]
정기 검증 (연 1회)

[검증 범위]
- 변별력 (KS, AUROC, Gini)
- 안정성 (PSI: 개발 vs 최근 12개월)
- 등급분포 단조성
- 캘리브레이션 (등급별 추정 PD vs CDR)

[입력 데이터]
- 위치: ./samples/credit_score_sample.csv (비식별 처리 완료)
- 컬럼: customer_id, obs_date, score, target, grade, set
- 표본 기간: 2022-01 ~ 2024-12
- 개발/운영 구분: set 컬럼 (dev / oot)

[표본 정보]
- 총 표본 수: 124,500
- 부도 건수: 1,872
- 등급 수: 10등급

[성공 기준]
- KS ≥ 0.30
- AUROC ≥ 0.70
- PSI < 0.25
- 등급별 부도율 단조성 위반 없음
- 보고서 10개 섹션 모두 작성

[제약 사항]
- 운영계 직접 접속 금지
- 외부 전송 금지
- 본 검증 결과는 인간 검증자의 최종 검토 후에만 의견서로 확정

[추가 메모]
- 챌린저 모형 (logistic) 비교 요청 여부: 예
```

---

## 요청 시 참고

- 데이터에 민감정보가 포함되지 않은지 사전 확인.
- 컬럼 정의가 모호한 경우, 오케스트레이터가 추가 질의할 수 있다.
- 표본 수가 임계 미만이면 결과 산출은 하되 강한 결론은 회피한다.
