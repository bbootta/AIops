# Data Definition Policy

본 하니스가 다루는 데이터의 정의·전제·금지 사항.

---

## 1. 데이터 카테고리

| 카테고리 | 정의 | 허용 |
|---|---|---|
| 합성 샘플 | 코드 내부에서 생성된 무작위 표본 | 분석/테스트 |
| 비식별 추출 | 마스킹·가명 처리된 검증용 추출 | 분석 |
| 운영 원천 | 실제 고객 식별정보 포함 가능 | 금지 |
| 외부 제공 | 거시 변수, 공개 통계 등 | 출처 명시 시 허용 |

운영 원천 데이터는 본 하니스 안에서 직접 다루지 않는다.

---

## 2. 표준 컬럼 명명

샘플/예시에서는 다음 명명을 우선한다 (실제 운영 스키마와는 무관).

- `customer_id`: 가명 식별자
- `obs_date`: 관측일
- `target`: 부도 여부 0/1
- `score`: 모형 점수 (높을수록 양호 또는 위험 — 모델별 명시)
- `pd`: 추정 PD (0~1)
- `lgd`: 추정 LGD (0~1)
- `ead`: 추정 EAD
- `stage`: IFRS 9 스테이지 1/2/3
- `scenario`: base / adverse / severe

---

## 3. 컬럼 정의 점검 의무

데이터를 받을 때 오케스트레이터는 다음을 확인한다.

- 컬럼별 타입 / 단위
- 결측 표기 방식 (NaN, NULL, 특수값)
- 시점 정의 (기준일, 관측구간)
- 목표변수의 정의 (90일 연체 여부, 12개월 부도 여부 등)
- 학습/검증 기간 분리 방식

확인되지 않은 컬럼은 분석에 사용하지 않는다.

---

## 4. 금지 사항

- 주민등록번호, 계좌번호, 카드번호, 전화번호, 이메일을 포함한 데이터 직접 처리
- 운영계 DB 직접 조회
- 데이터의 외부 전송
- 데이터를 `reports/`, `logs/`에 원본 그대로 기록 (요약·통계만 허용)
