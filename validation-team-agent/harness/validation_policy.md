# Validation Policy

검증 의견 도출의 원칙·필수 항목·작성 형식.

---

## 1. 검증 의견 유형 (참고)

- 적정
- 조건부 적정 (보완 후 적정)
- 부적정

본 하니스는 위 의견 중 어느 것도 **자동 확정하지 않는다**.
인간 검증자가 최종 확정한다. 본 하니스는 의견 **초안**만 제시한다.

---

## 2. 검증 의견 초안 작성 시 필수 항목

1. 검증 대상의 명확한 식별 (모형명/버전/사용 목적)
2. 적용된 검증 방법과 기준
3. 핵심 결과 수치
4. 결과의 한계와 추가 확인 사항
5. 의견 초안 (문안)
6. 인간 확인 요구 항목 (의견 확정 전 필수)

---

## 3. 작성 형식

```
[모형명/버전]
[검증 기간] / [표본 정보]
[검증 범위]

핵심 결과:
- 변별력: ...
- 안정성: ...
- 캘리브레이션: ...
- 시나리오/스트레스: ...

한계:
- ...

추가 확인 사항:
- ...

의견 초안:
"본 검증 결과 ... 의견을 제시함. 단, 본 의견은 초안이며 인간 검증자의
최종 검토와 승인 후에만 효력을 가짐."
```

---

## 4. 금지 행위

- 의견을 강하게 단정하는 표현 사용 (예: "확정적으로 적정함")
- 한계와 추가 확인 사항을 생략한 채 의견을 제시
- 표본 부족 상황에서 강한 결론 도출
- 검증 기준의 임의 완화
- 외부 제출본의 임의 확정
