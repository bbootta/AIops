# Skill — PD / LGD / EAD Validation

## 목적
신용위험측정요소의 추정 적정성·표본 적정성·캘리브레이션·안정성을 점검한다.

## 입력 (각 요소별)
- PD: 등급별 추정 PD, 등급별 부도 건수·표본 수
- LGD: 부도시점 LGD, 회수율 분포, 회수 시점 분포
- EAD: 한도 사용률, CCF 추정값, 부도시점 EAD

## 절차

### PD
1. 등급별 표본 수, 부도 건수 점검 (`middleware/sample_size_guard`)
2. CDR vs 추정 PD (`tools/metric_cdr_sdr.compare_cdr_sdr`)
3. binomial test로 캘리브레이션 유의성 (참고 - tools에 미구현 시 사용자에게 추가 분석 요청)
4. 등급별 부도율 단조성 점검
5. PSI로 등급분포 안정성

### LGD
1. 부도시점 표본 적정성 점검
2. 회수율 분포 (평균/중위/분위)
3. downturn LGD vs 평상시 LGD 비교 시점·표본 적정성
4. 추정 LGD vs 실측 LGD 비교

### EAD
1. CCF 산출의 표본 적정성
2. 한도 사용률 분포의 안정성 (PSI)
3. 추정 EAD vs 실측 EAD 비교

## 산출물
- 요소별 추정 vs 실측 비교표
- 표본 적정성 평가
- 안정성 (PSI)
- 한계와 추가 확인사항

## 금지
- 부도 정의를 임의로 변경
- 회수 인정 기준을 임의로 적용
- 등급체계의 실질적 변경 제안의 자동 확정

## 완료 기준
- 요소별 모두 표본 수, 추정 vs 실측, 안정성이 보고되었는가
- 표본 부족이 명시되었는가
