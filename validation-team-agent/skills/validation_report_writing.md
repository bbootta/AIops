# Skill — Validation Report Writing

## 목적
검증 결과를 검증보고서 초안 / 검증의견서 초안 / 점검표 / 요약 형태로 변환한다.

## 입력
- 분석 결과 dict (모형명, 표본 정보, 지표, 한계, 추가 확인사항)
- 보고서 양식 (없을 경우 `tools/report_template.build_validation_report` 기본 양식 사용)

## 절차
1. 결과 dict 검증 (필수 필드 존재 확인)
2. `tools/report_template.build_validation_report(result_dict)` 호출하여 초안 생성
3. `middleware/output_completeness_guard`로 필수 섹션·한계·추가 확인사항 누락 점검
4. 임의로 부드러운 표현 사용 금지 — 사실·근거 우선
5. 모든 수치에는 출처(파일명, 함수명) 명시

## 산출물 구조
표준 10개 섹션:
1. 요약
2. 검증 목적
3. 입력 데이터 및 전제
4. 검증 방법
5. 주요 결과
6. 이상 징후 및 원인 후보
7. 한계와 리스크
8. 검증 의견 초안
9. 추가 확인 사항
10. 감사추적 및 변경 이력

## 금지
- 외부 제출본 확정
- 검증 의견의 강한 단정
- 표본 부족 명시 누락
- 출처 없는 수치 인용

## 완료 기준
- 10개 섹션 모두 채워졌는가
- 누락 섹션이 없다는 점이 미들웨어 점검을 통과했는가
